The migration is primarily driven by platform lifecycle and vendor direction. The current WebSphere 8.x and RHEL-based infrastructure has reached end-of-life and is no longer aligned with enterprise support and security standards. In parallel, the vendor has transitioned their application roadmap to a Spring Boot–based runtime, with future versions no longer supporting WebSphere.
As a result, the application must be moved to a modern, supported platform. OpenShift has been selected as the target environment, providing a standardized, container-based platform aligned with enterprise architecture and operational practices.
